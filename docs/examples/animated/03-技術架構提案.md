<div class="tag">04 / REQUEST FLOW</div>

# 一次訂單，如何安全完成？

```mermaid {scale: 0.72}
sequenceDiagram
  actor U as User
  participant G as Gateway
  participant O as Order
  participant P as Payment
  participant E as Event Bus
  U->>G: POST /orders
  G->>G: Verify JWT + Rate Limit
  G->>O: Create Order
  O->>P: Authorize Payment
  P-->>O: Authorized
  O->>E: OrderCreated
  O-->>U: 201 Created
```

<div class="callout" v-click>同步路徑保持最短，其餘工作透過事件非同步完成。</div>

<div class="tag">05 / RELIABILITY</div>

# 失敗是預期，不是例外

<div class="reliability">
  <div v-click><b>Timeout</b><span>每個外部呼叫都有時間上限</span></div>
  <div v-click><b>Retry</b><span>指數退避與隨機抖動</span></div>
  <div v-click><b>Circuit Breaker</b><span>停止放大下游故障</span></div>
  <div v-click><b>Idempotency</b><span>重試不產生重複副作用</span></div>
  <div v-click><b>Dead Letter</b><span>保留失敗事件供後續處理</span></div>
  <div v-click><b>Tracing</b><span>跨服務追蹤每一次請求</span></div>
</div>
